## 各说明书内容大纲（统一模板）

每人按以下结构编写自己的模块说明书：

### 1. 模块概述
   - 模块功能描述
   - 解决的用户需求 / 业务流程中的位置

### 2. 数据库设计
   - 涉及的表及其结构（字段名、类型、约束）
   - 表之间的关系（ER 图用文字描述或 ASCII 图）
   - 关键 SQL 语句示例

### 3. 后端代码说明
   - **Model（实体类）**：字段对应、构造方法、关键方法
   - **DAO（数据访问层）**：核心方法及 SQL 逻辑
   - **Servlet（控制层）**：每个 Servlet 的 URL 映射、doGet/doPost 流程、请求参数、返回视图

### 4. 前端页面说明
   - JSP 页面功能、表单字段、交互逻辑
   - 页面跳转流程（附流程描述）

### 5. 核心业务流程（重点）
   - 用 **步骤序列** 描述完整流程
   - 示例：

     **例：用户注册流程**
     1. 用户在 `register.jsp` 填写表单
     2. 表单 POST 到 `RegisterServlet`
     3. Servlet 调用 `UserDao.register()` 验证字段 → 插入数据库
     4. 成功 → 重定向到 `login.jsp`；失败 → 返回注册页带错误信息

### 6. 关键代码片段
   - 每个模块选 3-5 个核心方法，粘贴代码并写注释说明

### 7. 测试用例
   - 正常流程测试（happy path）
   - 异常处理测试（空字段、重复用户名、未登录访问等）

---

## 写作要求

- **语言**：中文
- **格式**：Markdown（`.md` 文件）
- **存放位置**：`Freelite/docs/说明书_模块名.md`
- **字数**：每份 1500-3000 字（含代码片段）
- **协作方式**：每人写完后 PR 到主仓库，组长review合并
